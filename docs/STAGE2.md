# Этап 2: Формирование машинного кода

## Цель
Реализовать логику преобразования команд в их двоичное представление.

## Выполненные задачи

### 1. Реализован транслятор из промежуточного в машинное представление
- Добавлен метод `encode()` в класс `Command`
- Реализовано кодирование для всех типов команд:
  - `LOAD_CONST` (158) - 6 байт
  - `READ_MEM` (17) - 5 байт  
  - `WRITE_MEM` (12) - 3 байта
  - `ABS` (214) - 5 байт

### 2. Реализована запись результата ассемблирования в двоичный файл
- Создан класс `Encoder` для кодирования списка команд
- Добавлена запись бинарных данных в выходной файл

### 3. Реализован вывод байтового представления в режиме тестирования
- Добавлен метод `to_hex_string()` для hex-представления команд
- Добавлен метод `to_test_format()` для представления в формате спецификации
- В режиме `--test` выводится как промежуточное, так и байтовое представление
